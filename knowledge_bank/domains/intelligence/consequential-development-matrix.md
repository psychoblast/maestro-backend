# Consequential Development Matrix (CDM)

After an item passes the Four-Filter Method ([[decision-change-methodology]]), it is
classified by *consequence level* — which determines whether it triggers an immediate
alert or enters the regular cycle, and the minimum source tier required. Classification
is based on **decision impact to specialists, not on how newsworthy the item is.**

---

## 1. The Classification Matrix

| Class | Name | Definition | Timing | Source Floor |
|-------|------|-----------|--------|--------------|
| **CDM-1** | Rule-Changer | Changes an existing rule every specialist in that domain relies on | Immediate alert | Primary, confirmed |
| **CDM-2** | Strategy-Invalidator | Makes a currently active or recommended strategy suboptimal or wrong | Immediate alert | Primary, or two strong-trade sources |
| **CDM-3** | Opportunity-Creator | Opens a significant new actionable window | Regular cycle (urgent framing) | Strong trade |
| **CDM-4** | Contextual-Update | Updates the picture without invalidating any current strategy | Regular cycle (standard) | Strong trade or useful signal |
| **CDM-5** | Monitoring Signal | Not yet actionable; warrants watching | Regular cycle (monitoring flag) | Useful signal acceptable |

**Classification anti-pattern:** treating a "significant headline" as automatically
CDM-1 or CDM-2. Newsworthiness is not consequence.

**Practitioner insight — the most common misclassification is CDM-2.** The temptation
is to treat any big news as CDM-1. The discipline: does this change a *rule*
(CDM-1), or does it change what the *right answer is within the existing rules*
(CDM-2)? A platform raising a minimum-stream-length requirement changes a rule; a
territory's streaming growth halving changes the right answer inside unchanged rules.

---

## 2. Worked Distinctions

| Development | Classification | Reasoning |
|-------------|----------------|-----------|
| A major streaming platform raises its minimum-payable-stream-length requirement globally | CDM-1 Structural | The rule of the game changes; affects everyone with payout or revenue implications |
| A platform's editorial team is reorganized — several playlist editors move between genres | CDM-4 Contextual | Personnel change affects the contact map; no rule or strategy framework changes |
| A territory's streaming growth slows to 2% year-on-year from 8% prior (primary data) | CDM-2 Strategy-Invalidator | Changes revenue projections for every streaming-dependent strategy; warrants urgent framing |
| A single act charts at #47 in one territory | Noise | One act's chart position; no structural implication — dropped at Filter 3 |
| A dominant regional platform acquires a competitor; consolidation confirmed | CDM-1 Structural | Market-structure change reshapes routing and strategy across that territory |
| A regional platform launches in two new countries | CDM-3 Opportunity-Creator | New coverage creates a specific window for specialists routing to that region |
| A trade outlet reports a platform "may be considering" a tier change (single weak source) | CDM-5 Monitoring Signal | Insufficient sourcing for CDM-1/2; flag and monitor |

---

## 3. From Classification to Timing

Classification sets the *default* timing; sourcing can hold it back:

- **CDM-1 / CDM-2** route to the immediate-alert pathway — but only fire once sourcing
  clears the floor. A CDM-1 on a single strong-trade source is held 24–48h for a
  second source and downgraded if none arrives.
- **CDM-3** is written into the regular cycle with urgent framing and led in its
  section.
- **CDM-4** is a standard regular-cycle entry.
- **CDM-5** carries minimum detail in a monitoring section with a named re-check
  trigger.

quiet-cycle protocol — lives in [[triage-and-routing]]. Anti-fabrication rules that
