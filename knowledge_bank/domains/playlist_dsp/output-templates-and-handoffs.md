# Playlist and DSP — Output Templates and Handoffs

Three deliverable types. Every one carries the **mandatory core**; conditional modules attach
per the matrix only (anti-bloat — a curator email draft does not ship with a risk register).

## Mandatory core (all types)

1. **Work Product** — the deliverable in the type's format below.
2. **Decision Summary** — recommendation + reasons as an executive memo, answering: *Why? Why
   now? What data supports this? What assumptions am I making? What would change my answer?*
3. **Confidence Assessment** — level + reasons + risks, capped by the weakest critical input
   with the link named (e.g., "capped at low-medium: target list health inferred from public
   signals only").

## Conditional-module matrix

| Module | Pitch Package | Strategy Memo | Post-Placement Read |
|---|---|---|---|
| Scores (P-verdicts) | ✅ P1–P4 always (P5 NOT_EVALUABLE pre-add) | ◐ when ranking targets | ✅ P5 + revisited P1 |
| Cost & ROI | — (any spend ⇒ escalate to owner first) | ◐ only if spend proposed | — |
| Opportunity Ranking | ◐ when >1 target compared | ✅ portfolio ranked Impact/Effort/Probability-band | — |
| Risk Register | — (a pitch draft never carries one) | ◐ strategic commitments only | — |
| Prediction Log Entry | ✅ always — every pitch is a logged fit prediction | ✅ for forecastable claims | ◐ when the read implies a next-placement expectation |
| Memory Update | ✅ contacts + playlists (evidence-tagged) | ◐ | ✅ placement history + curator account update |
| Alternatives | ✅ | ✅ | ✅ |
| Next Best Action | ✅ | ✅ | ✅ |

## Type 1 — Pitch Package

**Use:** a specific track aimed at specific target(s). Refused outright if the fit gate fails or
a pay-for-play red flag exists (a refusal, not a deduction). **When refusing for any reason, a
constructive next step is mandatory** — name what to fix (re-time the pitch, address the fit
gap, cure the account damage) and how to improve for the next attempt. A bare refusal is a named
failure mode.

```
PITCH PACKAGE — [track] → [target list/curator] · [economy: editorial|independent] · [date]
BAND B GATES        [fit-token gate · pitch-quality bars · cadence/quota · timing gate · target-health vet — pass/fail/not-evaluable, each w/ source]
FIT CASE (P1)       [verdict + named fit rationale: sonic / audience / timing — evidence-tagged; inferred preferences marked as hypotheses]
RELATIONSHIP (P4)   [state w/ evidence (observed/told/inferred) · trust-account read · why this ask is affordable now]
THE PITCH           [draft: subject <8 words · ≤3-paragraph who/fit/ask body · one ask · no flattery]
TIMING (P3)         [window math: release date, pitch-by date, follow-up date per tier cadence]
EXPECTATION         [placement-probability band IN WORDS w/ reasons — never a %, never a guarantee · post-add P5 check plan]
DECISION SUMMARY    [the five questions]
CONFIDENCE          [level + reasons + risks + named weakest link]
ALTERNATIVES        [one credible alternative + why not chosen, or "none credible — [reason]"]
NEXT BEST ACTION    [one concrete step in 24–72h]
PREDICTION LOG      [entry path · falsifiability: add/no-add by (window) · post-add retention expectation]
MEMORY UPDATE       [contacts/playlists entries w/ tags]
```

## Type 2 — Playlist Strategy Memo

**Use:** "how do we approach playlisting for [artist/release/quarter]?"

```
PLAYLIST STRATEGY MEMO — [subject] · [horizon] · [date]
CURRENT POSITION    [Band A facts: current placements, source mix, owned-audience baseline]
THE THREE GAMES     [per economy: position, move, and what we are NOT doing and why]
TARGET PORTFOLIO    [Opportunity Ranking: targets w/ Impact / Effort / Probability-band; concentration-risk note]
SEQUENCE            [cross-economy sequence w/ the assumption each step rests on; sequence claims = logged predictions]
GUARDRAILS          [what this strategy refuses (schemes, unfit reaches) — stated before a vendor proposes them]
DECISION SUMMARY    [the five questions]
CONFIDENCE          [level + reasons + risks + named weakest link]
ALTERNATIVES        [one credible alternative strategy + why not chosen, or "none credible — [reason]"]
NEXT BEST ACTION    [one concrete step in 24–72h]
[modules per matrix]
```

## Type 3 — Post-Placement Read

**Use:** ≥7 days after an add (consumes cohort discipline from the data and analytics
discipline; below the window: facts only). **Below-window protocol (< 7 days post-add):** report
the mandatory early-read fact set — playlist/chart position + day-1/2 exposure — render P5 as
NOT_EVALUABLE(window), and schedule the full read with its date. Do not issue conversion
verdicts on sub-window data.

```
POST-PLACEMENT READ — [track] on [list] · [add date → read date]
BAND A              [placement-cohort metrics w/ definitions deferred to the analytics discipline; position/time-on-list]
P5 VERDICT          [retention signal: converting / coasting / failing — verdict + evidence; NO benchmark-band numbers until the unlock]
P1 REVISITED        [was the fit call right? hit/miss vs. the logged prediction — honest scoring of our own pitch]
ACCOUNT EFFECTS     [trust-account deposit/withdrawal · removal post-mortem if removed (refresh vs. under-performance vs. list death)]
NEXT MOVE           [keep pitching this lane / fix before next pitch / stop]
DECISION SUMMARY    [the five questions]
CONFIDENCE          [level + reasons + risks + named weakest link]
ALTERNATIVES        [one credible alternative next move + why not chosen, or "none credible — [reason]"]
NEXT BEST ACTION    [one concrete step in 24–72h]
PREDICTION RESOLUTION [outcome entry path for the original pitch prediction — wins/losses/surprises routing]
MEMORY UPDATE       [placement history + curator account, evidence-tagged]
```

A bare data pull or a "who covers this genre" lookup is none of these types: facts with sources,
no modules, no memo.

## Handoff boundaries and escalation flags

- **Agent-to-agent handoffs** are band-labeled with verdicts, evidence tags, and the named
  confidence cap.
- **Route out, do not advise:** legal/rights and ToS questions → legal and business affairs;
  metric definitions and statistical reads → the data and analytics discipline; any spend → the
  owner before commitment.
- **Escalate to a human:** any reply or situation flagged for human handling, first contact with
  named platform editorial staff outside the official channel, and any continuation of a
  relationship after a pay-for-play signal.
