# Playlist and DSP — Scoring Rubric

Three epistemic bands, never blended: **MEASURED / CHECKED / JUDGED.** Quality bars come before
scores; numbers without outcome data ship as verdicts with named caps, not invented figures.

## Reconciliation principles

1. **Quality bars before scores** — pitch quality is enforced as hard pass/fail rules, not soft
   numbers. Checkable bars live in the CHECKED band; judgment lives in the JUDGED band.
2. **Outcomes re-rank** — observed response behavior outweighs static tier over time; accuracy
   data plays the same role at the judgment layer.
3. **No unanchored numbers** — dimensions without outcome data ship as verdicts with named caps;
   numeric projections are deferred behind an outcome-volume unlock (see below).
4. **Three epistemic bands** — MEASURED / CHECKED / JUDGED, kept separate.

## Band A — MEASURED (facts)

Track metadata and audio facts (consumed from the data and analytics discipline where
available); curator-record fields (tier, genres, notes, last-pitched date, response rate — each
with its evidence tag: response rate is observed; notes are told/inferred); and platform-rule
facts from the intelligence feed, each carrying a source-trust tier.

## Band B — CHECKED (pass / fail / not-evaluable, each check citing its source)

| Check set | Checks |
|---|---|
| **Fit gate** | every genre token of the track matches the target's coverage (tokenized, all-tokens rule) |
| **Pitch-quality bars** | subject under ~8 words & outlet-specific · body ≤ 3 paragraphs in who/fit/ask structure · a named THIS-curator fit rationale present · exactly one ask · no filler/flattery |
| **Cadence & volume** | follow-up timing per tier (A 3d / B 5d / C 7d) · daily send ceiling (~50/day) · per-curator cadence respects relationship state |
| **Timing gate** | editorial pitch submitted within the platform window · release-assets checklist complete pre-send |
| **Target-health vet** (independents) | engagement plausibility vs. follower count · churn pattern · pay-for-play red flags (a public fee-for-add ⇒ disqualified) |

A failed fit gate or a pay-for-play red flag **blocks the pitch** — a refusal, not a deduction.
These are **hard blocks: non-overridable.** A WEAK judged playlist-fit verdict on an otherwise
gate-passing pitch is a **soft block**: the send pauses by default, the specific fix is stated,
and it may proceed only on an explicit artist override.

## Band C — JUDGED (closed verdict vocabulary: STRONG / PROMISING / MIXED / WEAK / NOT_EVALUABLE(reason))

| # | Dimension | What it judges |
|---|---|---|
| P1 | **Playlist fit** | Beyond the token gate: sonic/audience/intent fit with *this* list's programming contract, using evidence-tagged curator notes and list-behavior observation. The primary pitch-quality dimension. |
| P2 | **Pitch quality** | The persuasive residual after the Band B bars pass: does the fit rationale actually reduce the gatekeeper's risk; is the proof hierarchy used well? |
| P3 | **Timing** | Calendar fit: platform window, list-refresh rhythm, competitive release context, artist readiness. |
| P4 | **Curator-relationship state** | cold/opened/warm/damaged/closed with evidence; whether *this ask, now* is affordable from the trust account. |
| P5 | **Post-placement retention signal** | After an add: does the placement convert (save rate by source, follow conversion, skip behavior — definitions consumed from the data and analytics discipline)? Pre-placement and pre-data: **NOT_EVALUABLE**. When NOT_EVALUABLE(window) applies (add < 7 days ago), the mandatory early-read fact set is playlist/chart position + day-1/2 exposure. |
| S | **Specificity** | PASS/FAIL only. PASS: names a specific sonic, audience, or timing fit the curator can verify in seconds. FAIL: generic language, superlatives, or "you'll love this" with no checkable claim. |

Every P-verdict carries confidence with reasons and risks, capped by the weakest critical input
with the link named (e.g., "P1 STRONG capped at medium confidence: the list's programming
contract is inferred from public behavior, not stated by the curator"). Inferred curator
preferences never push a verdict above PROMISING on their own.

## Placement-probability language

Placement probability is expressed **only as a banded qualitative read with reasons** (e.g.,
"strong editorial case, band: plausible-to-likely"), never as a percentage, and never as a
guarantee. Numeric probability bands unlock under the same condition as the numeric layer below.

## Numeric layer — deferred, with the unlock condition

No numeric save-rate projections, fit scores, or placement percentages until an outcome corpus
of resolved pitch predictions (logged pitches resolved against adds + post-placement reads) is
large enough to anchor bands — a working floor of roughly ≥30 resolved predictions per economy
(editorial / independent), plus an evidence-cited change record. Until then: verdicts,
bands-in-words, and named caps only. Post-unlock changes require outcome evidence; counterfactual
reasoning never counts as evidence.
