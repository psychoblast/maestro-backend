# Catalog Health Rubric — PLMKR Music Publishing

The ten-dimension Catalog Health Model Reed uses to assess the publishing and rights
state of a song, a writer's catalog, or an acquisition target. The composite is
**PROVISIONAL** — secondary to the per-dimension grades — until an outcome corpus of
≥30 outcome-checked catalog evaluations exists.

---

## Provisional Composite Discipline

Until the unlock condition (≥30 outcome-checked evaluations) is met:
1. Score each dimension A+–F with per-dimension justification citing evaluable sub-signals.
2. Calculate the provisional composite (formula below).
3. Report it labeled **PROVISIONAL COMPOSITE**.
4. State the unlock condition in every output: *"PROVISIONAL — unlock condition: ≥30 outcome-checked evaluations."*
5. The provisional composite is secondary to per-dimension grades and is not presented as calibrated or comparable across catalogs.

---

## Asset-Recovery Frame (Answer Explicitly in Every Evaluation)

> "If this catalog were being prepared for sale, licensing, or audit today, what
> would the due-diligence process find — and what would it discount the value by?"

Answer it with evidence: which dimensions create discount risk, which create upsell
opportunity, which are genuinely strong. If the answer requires hedging, state the
specific data that would resolve the uncertainty. Every dimension is assessed through
the asset-management lens: what is the economic consequence of its current state, and
what specific action would capture the value being left on the table?

## Ten-Dimension Catalog Health Model

| # | Dimension | Weight | Asset-Management Connection |
|---|-----------|--------|------------------------------|
| 1 | **Registration Completeness** | **0.15** | Foundation: unregistered works cannot collect |
| 2 | **Collection Coverage** | **0.15** | Revenue: all income streams actively collecting |
| 3 | **Royalty Recovery Readiness** | **0.13** | Actionable: quantifiable value currently uncaptured |
| 4 | **Identifier Completeness** | **0.12** | Infrastructure: ISWC/IPI prerequisite for matching |
| 5 | **Ownership Clarity** | **0.12** | Prerequisite: disputed ownership blocks licensing/transactions |
| 6 | **Licensing Readiness** | **0.12** | Revenue activation: catalog ready for rapid commercial use |
| 7 | **Territorial Coverage** | **0.08** | Scale: collecting where usage occurs |
| 8 | **Metadata Quality** | **0.07** | Precision: statement matching and routing accuracy |
| 9 | **Audit Status** | **0.04** | Protection: accounting accuracy preserved within window |
| 10 | **Legal Exposure** | **0.02** | Risk: active litigation or uncleared exposure discounting value |
| | **Total** | **1.00** | |

Weights are tagged **[PLMKR-DEFAULT]** pending outcome calibration.

## Anti-Fake-Precision Mechanics

Each sub-signal is classified MEASURED / SOURCED / JUDGED (see publishing-doctrine.md
§ Source Tiers). Grading rules:
1. Grade a sub-signal only where evidence exists for that signal type.
2. If no evidence exists: mark **NOT EVALUABLE** and name the minimum data required.
3. Each dimension receives a letter grade based only on evaluable sub-signals.
4. The dimension assessment states: share of sub-signals evaluated, share NOT EVALUABLE, and which inputs drove the grade.
5. **Dimension confidence is capped by the weakest critical sub-signal — named explicitly.**

**Probability language rule:** confidence is qualitative (HIGH / MEDIUM / LOW with
reasons); numeric probability percentages are forbidden until the unlock condition is
met.

**Letter grade → numeric equivalent:**

| Grade | Numeric | Grade | Numeric |
|-------|---------|-------|---------|
| A+ | 10.0 | C+ | 6.0 |
| A | 9.0 | C | 5.0 |
| A– | 8.5 | C– | 4.5 |
| B+ | 8.0 | D+ | 4.0 |
| B | 7.0 | D | 3.0 |
| B– | 6.5 | D– | 2.5 |
| — | — | F | 1.0 |

## D1 — Registration Completeness (0.15)

Are all compositions registered with the appropriate PROs and mechanical
organizations in every territory where income is (or could be) collected?

Sub-signals: PRO registration count vs. catalog count (home territory, MEASURED);
mechanical registration completeness — The MLC, MCPS, etc. (MEASURED); new-works
registration lead time (SOURCED+JUDGED); historical registration gap analysis
(SOURCED+JUDGED); black-box losses traceable to missing registrations (SOURCED).

Anchors [PLMKR-DEFAULT]: **A** = ≥98% home-PRO registered, ≥95% mechanical, new works
registered before/concurrent with every DSP delivery, no known black-box losses,
historical gaps resolved. **B** = ≥90% PRO / ≥85% mechanical, process in place with
documented timing gaps. **C** = 70–89% coverage, no systematic new-works process,
some black-box returns, gaps not yet analyzed — registration reactive. **D and below**
= <70% PRO coverage, widespread mechanical gaps, multiple confirmed black-box losses.
**F** = active catalog essentially unregistered. Confidence is PARTIAL absent
registration-audit or society-portal data.

## D2 — Collection Coverage (0.15)

Is the catalog actively collecting all applicable streams — performance, mechanical,
neighboring rights, digital/streaming, sync, UGC — across the territories where usage
occurs?

Sub-signals: active income streams vs. applicable streams (SOURCED); territory
collection infrastructure (SOURCED+JUDGED); CMS/UGC monetization status
(SOURCED+JUDGED); neighboring-rights infrastructure (SOURCED); black-box proactive
claim history (JUDGED).

Anchors [PLMKR-DEFAULT]: **A** = all applicable streams collecting, UGC deployed and
verified, NR claims registered, sub-pub/affiliations covering all active territories,
proactive black-box practice. **B** = performance + mechanical fully collecting, ≥1
UGC platform monetized, NR initiated, sub-pub in top territories. **C** = performance
and mechanical in home territory only, no UGC, NR not pursued, foreign royalties via
reciprocal only. **D and below** = mechanical gaps, no UGC, no NR, no international
infrastructure — passive, home-territory only. Confidence PARTIAL absent multi-stream
statement history.

## D3 — Royalty Recovery Readiness (0.13)

Is the catalog positioned to identify and recover royalties owed but uncollected —
unmatched distributions, retroactive registration recovery, historical black-box
claims, foreign uncollected pools?

Sub-signals: MLC unmatched-pool claims filed — US (SOURCED); black-box claim strategy
and history (JUDGED); retroactive registration recovery analysis (SOURCED+JUDGED);
foreign society uncollected-pool assessment (SOURCED+JUDGED); unmatched royalty
history in statements (MEASURED).

Anchors [PLMKR-DEFAULT]: **A** = MLC claims filed where applicable, active society
claim program, retroactive windows tracked and filed, foreign claims current,
unmatched items disputed. **B** = MLC claims filed/in process, some international
claims, retroactive gaps identified, statement items reviewed annually. **C** = no MLC
claims, no international program, gaps identified but no action. **D and below** = no
awareness or action on any recovery mechanism. Recovery estimates ONLY where evidence
supports them, labeled ESTIMATE with basis.

## D4 — Identifier Completeness (0.12)

Does every active work have complete, accurate, linked identifiers enabling automated
matching across global collection, streaming, and licensing systems?

Sub-signals: ISWC issuance/assignment rate (MEASURED); IPI registration — all writers
(SOURCED); IPI registration — all publishers (SOURCED); ISRC linkage to masters
(SOURCED); IPI deduplication / name-variant accuracy (SOURCED+JUDGED); variant-title
documentation (SOURCED+JUDGED).

Anchors [PLMKR-DEFAULT]: **A** = ≥98% valid ISWC, all writers/publishers with IPI,
ISRC linkage complete, no IPI duplicates, variant titles registered. **B** = ≥90%
ISWC, most IPI present, some ISRC gaps on older catalog. **C** = 70–89% ISWC, some
co-writer IPI gaps, partial ISRC, variant titles unregistered. **D and below** = <70%
ISWC, multiple works with unregistered writers/publishers. **F** = active
revenue-generating works with neither ISWC nor PRO registration.

**Hard Gate HG-1 — Active Revenue Without Identifiers:** a work with confirmed active
streaming/broadcast revenue but no ISWC and no PRO registration collects zero
performance income and matches zero mechanical. Such works score **D** for this
dimension regardless of other infrastructure quality.

## D5 — Ownership Clarity (0.12)

Is beneficial ownership of every work unambiguous, documented with executed
agreements, and defensible against a third-party challenge?

Sub-signals: split-sheet documentation for co-written works (SOURCED); work-made-for-
hire documentation (SOURCED); chain of title for transfers/acquisitions
(SOURCED+JUDGED); active ownership disputes (SOURCED); producer/co-producer split
clarity (SOURCED+JUDGED); controlled-composition clause interaction (SOURCED+JUDGED).

Anchors [PLMKR-DEFAULT]: **A** = all co-written works with executed splits, WFH
documentation complete, full chain of title, zero active disputes, producer
arrangements executed — defensible to a due-diligence standard. **B** = ≥90% executed
splits, recent chain of title documented, isolated legacy questions. **C** = 70–89%
executed splits, some chain-of-title gaps, 1–3 active questions under resolution.
**D and below** = material split-sheet gaps, multiple chain-of-title issues, active
disputed claims.

A disclosed-but-unfiled ownership dispute that does not trigger HG-2 must still be
surfaced and documented as a flagged ownership risk. Confidence PARTIAL when split
sheets / chain of title are NOT EVALUABLE.

## D6 — Licensing Readiness (0.12)

Is the catalog packaged and cleared for commercial licensing — particularly sync and
brand use — in a form that enables rapid execution?

Sub-signals: one-stop status, composition + master (SOURCED+JUDGED); sync clearance
infrastructure (JUDGED); master recording rights clarity (SOURCED); publisher sync
representative (SOURCED+JUDGED); license fulfillment turnaround (JUDGED); catalog
searchability and pitch readiness (JUDGED).

Anchors [PLMKR-DEFAULT]: **A** = one-stop clearance for core catalog, rate card with
media-tier ranges, license template current, single responsive sync contact, master
rights clearly controlled, pitch-ready — executable within 24 hours. **B** = two-party
clearance with both rightsholders identified and cooperative, pricing guidelines,
sync rep in place — 2–5 business days. **C** = ad hoc negotiation each time, no rate
card, no designated contact. **D and below** = infrastructure absent, master rights
unclear or held by an unresponsive party.

**Hard Gate HG-2 — Unresolved Ownership Dispute:** a work with an unresolved active
ownership claim caps at **D+** for Licensing Readiness regardless of infrastructure
quality. A disputed work cannot be reliably licensed. HG-2 requires a formally filed
PRO conflict or legal claim on record. Confidence PARTIAL when one-stop status /
master rights clarity are NOT EVALUABLE.

## D7 — Territorial Coverage (0.08)

Is the catalog generating income in all territories where meaningful usage occurs —
via direct PRO affiliation, sub-publishing, or confirmed reciprocal coverage?

Sub-signals: revenue territory vs. usage territory overlap (SOURCED+MEASURED);
sub-publisher coverage in non-domestic territories (SOURCED); reciprocal coverage in
streaming territories (SOURCED+JUDGED); territory-specific registration compliance —
JASRAC, KOMCA, ECAD, etc. (SOURCED+JUDGED); international collection-timeline
management (SOURCED+JUDGED, [Tier C: typically 6–24 months]).

Anchors [PLMKR-DEFAULT]: **A** = top-10 territories confirmed via statements, sub-pub
or direct affiliation in all material territories, reciprocal registration confirmed,
lag tracked. **B** = top-5 territories collecting, some direct affiliations, lag
managed. **C** = home territory + 2–3 majors via sub-pub, no direct non-Western
affiliations. **D and below** = home territory only, no international infrastructure,
material usage generating zero royalties.

## D8 — Metadata Quality (0.07)

Are registered works complete and consistent — correct writer shares, publisher
participation, ISWC/ISRC linkage, variant-title resolution — enabling accurate
statement matching and routing?

Sub-signals: statement matching rate (MEASURED); writer-share consistency across
systems (SOURCED+JUDGED); ISWC linkage in CMS (SOURCED); variant-title registration
(SOURCED+JUDGED); publisher participation accuracy (SOURCED).

Anchors [PLMKR-DEFAULT]: **A** = matching rate ≥95%, shares consistent across all
systems, ISWC linked, variant titles registered. **B** = 85–94% matching, minor
discrepancies under reconciliation. **C** = 70–84% matching, known discrepancies
unresolved, partial ISWC linkage. **D and below** = <70% matching, systematic errors
generating material unmatched royalties.

## D9 — Audit Status (0.04)

Is the catalog actively monitored for accounting accuracy — and are audit rights
exercised before contractual windows close?

Sub-signals: last audit date vs. contractual window (SOURCED); prior audit recovery
history (SOURCED+JUDGED); statement-review practice (JUDGED); new-revenue-stream
monitoring (JUDGED); audit-clause status in current agreement (SOURCED).

Anchors [PLMKR-DEFAULT]: **A** = last audit within window, annual review practice, new
streams monitored, audit clause understood. **B** = audit within window, irregular
review, window not approaching expiry. **C** = no recent audit, ad hoc review, window
clock running. **D and below** = window expired/expiring without action, no review.

**Hard Gate HG-3 — Expired Audit Window:** if the contractual audit window has passed
without an audit, Audit Status scores **D** regardless of other factors. Waiver of
audit rights is a permanent loss.

## D10 — Legal Exposure (0.02)

Does the catalog carry material legal risk — active litigation, uncleared sample
exposure, pending contractual disputes — that could reduce value or create liability?

Sub-signals: active infringement claims naming the catalog (SOURCED); uncleared
sample exposure (SOURCED+JUDGED); termination-risk window approaching
(SOURCED+JUDGED); contractual compliance status (SOURCED+JUDGED); AI training-
licensing exposure (JUDGED — labeled CONTESTED, no settled law).

Anchors [PLMKR-DEFAULT]: **A** = no active litigation, no known uncleared samples,
termination windows tracked, agreement in good standing, AI exposure assessed with a
contestedness label. **B** = no active litigation, minor sample risk in catalog tail.
**C** = one resolved/settling matter, sample exposure not remediated, windows not
tracked. **D and below** = active litigation, material uncleared sample exposure in
active works, termination windows unmanaged.

**Hard Gate HG-4 — Active Litigation:** a catalog with active copyright-infringement
litigation naming it scores **D** for Legal Exposure until the litigation resolves,
and the matter is prominently disclosed in every evaluation output.

## Hard Gates Summary

| Gate | Dimension | Trigger | Grade Floor |
|------|-----------|---------|-------------|
| **HG-1 Active Revenue Without Identifiers** | D4 | Active revenue, no ISWC, no PRO registration | D for affected works |
| **HG-2 Unresolved Ownership Dispute** | D6 | Active, formally filed ownership claim | D+ cap for Licensing Readiness |
| **HG-3 Expired Audit Window** | D9 | Contractual window passed without audit | D forced for Audit Status |
| **HG-4 Active Litigation** | D10 | Active infringement litigation naming the catalog | D forced for Legal Exposure |

Gates are dimension-level, not composite-level. A triggered gate degrades the relevant
dimension; it does not override the composite or produce a PASS verdict. The composite
reflects the triggered grade naturally.

## Composite Formula (PROVISIONAL — Secondary to Per-Dimension Grades)

```
Composite = (RC × 0.15) + (CC × 0.15) + (RRR × 0.13) + (IC × 0.12)
          + (OC × 0.12) + (LR × 0.12) + (TC × 0.08) + (MQ × 0.07)
          + (AS × 0.04) + (LE × 0.02)
```

Each dimension value is the numeric equivalent of its letter grade. Maximum possible
composite: 10.0. Weights sum: 1.00 (0.15+0.15+0.13+0.12+0.12+0.12+0.08+0.07+0.04+0.02).
Label every output: *"PROVISIONAL COMPOSITE — not calibrated; not comparable across
catalogs."*

## Action Profile (Primary Actionable Output)

Per-dimension grades generate a prioritized remediation queue based on dimension
grade × weight × recoverability estimate.

| Action Tier | Trigger | Meaning |
|-------------|---------|---------|
| **IMMEDIATE** | Any dimension at D+ or below | Active value loss or risk; remediate first |
| **PRIORITY** | Any dimension at C / C– weighted ≥0.10 | Significant value on the table; structured plan |
| **OPTIMIZE** | Any dimension at C+ / B– weighted ≥0.10 | Improvement available; next operating cycle |
| **MAINTAIN** | Dimension at B or above | Protect and monitor; no active gap |
| **EXEMPLARY** | All dimensions at A– or above | Best-in-class benchmark |

Output order: IMMEDIATE actions (dimension, gap, specific action, estimated
recovery/risk reduction); PRIORITY (ordered by weight × severity); OPTIMIZE (ordered
by effort-to-impact); MAINTAIN (listed with grade); one-sentence catalog-health
summary from the asset-management posture.

## NOT QUOTABLE Discipline

When a client provides confidential-by-structure terms (publisher advance amounts,
above/below-statutory rates, audit settlements, net-publisher's-share percentages,
co-pub/admin fee rates from named deals): use them for analysis, label the analysis
*[CLIENT-SPECIFIC — NOT QUOTABLE in external output]*, use only published industry
ranges (Tier B/C labeled) when estimating comparables, and redact all client-specific
terms before any third-party distribution.

---

## Required Output Elements (Every Catalog Health Evaluation)

1. Per-dimension assessment — letter grade + evaluable sub-signals + NOT EVALUABLE sub-signals + confidence + what drove the grade + hard-gate check.
2. Asset-Recovery Frame answer — what due diligence would find and discount by, with evidence.
3. Action Profile — IMMEDIATE / PRIORITY / OPTIMIZE / MAINTAIN tiers.
4. PROVISIONAL composite — numeric, labeled, with the unlock condition verbatim.
5. Owner input gaps — items that would change the evaluation if resolved.
6. Confidence statement — overall confidence, naming the weakest critical input.

Add the NOT QUOTABLE flag if client-specific terms were provided.
