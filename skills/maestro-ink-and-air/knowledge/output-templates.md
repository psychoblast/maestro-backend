# Output Templates — PLMKR Music Publishing

Five output formats Reed uses to structure publishing work product. Each names what is
NOT EVALUABLE, never fabricates figures without a Tier A/B source, and routes
out-of-scope work to the right PLMKR specialist. Use the template that matches the
request; do not force a structured template onto a quick conversational answer.

---

## Template 1 — Catalog Health Evaluation (Flagship)

The full ten-dimension assessment (catalog-health-rubric.md). Two sections.

**Section A — State of the Catalog**
1. Context line: catalog / owner / scope, and whether this is a fixture or production assessment.
2. Ten-dimension assessment table: for each dimension — letter grade, weight, evaluable sub-signals named, NOT EVALUABLE sub-signals named, confidence (HIGH/MEDIUM/LOW with reason), what drove the grade, and the hard-gate check.
3. Hard-gate status (HG-1..HG-4): CLEAR or TRIGGERED with reason.
4. Owner input gaps: items that would change the evaluation if resolved.

**Section B — What to Do**
1. Action Profile: IMMEDIATE / PRIORITY / OPTIMIZE / MAINTAIN tiers, each with dimension, gap description, specific action, and estimated recovery/risk reduction (labeled ESTIMATE with basis, or NOT ESTIMABLE).
2. Asset-Recovery Frame answer: what due diligence would find and discount by, with evidence.
3. Decision summary: one-paragraph state of the asset from the asset-management posture.
4. PROVISIONAL composite: numeric value + formula + label *"PROVISIONAL — unlock condition: ≥30 outcome-checked evaluations"*.
5. Confidence assessment: overall confidence, naming the weakest critical input that caps it.
6. Alternatives, Next Best Action (24–72h), Scope-Fence Handoffs, Risk Register.

Add the NOT QUOTABLE flag if client-specific terms were provided.

## Template 2 — Revenue Leak Report

Diagnoses lower-than-expected royalties against the nine-category leak taxonomy
(publishing-doctrine.md §6) before concluding there is a market or usage problem.

1. Context line.
2. Nine-category leak diagnostic: for each category — present / not present / NOT EVALUABLE, the detection signal observed, and the affected income stream.
3. Leak priority summary: ordered by estimated recoverable value × confidence (recoverability ESTIMATE only where evidence supports it, with stated basis; otherwise NOT ESTIMABLE).
4. Decision summary, confidence assessment, alternatives, Next Best Action (24–72h).
5. Scope-Fence Handoffs (financial modeling → Finance/Royalties; legal recovery action → counsel) and Risk Register.

Anti-pattern guard: never conclude "the stream isn't generating revenue" from a missing
statement line item. Work back through infrastructure → registration → identifier →
timeline first. A missing royalty line is most often an infrastructure failure, not a
usage failure.

## Template 3 — Publishing Deal Evaluation

Evaluates a deal structure against writer-vs-publisher economics (publishing-doctrine.md
§7). Identifies and explains; never advises on negotiation execution.

1. Context line: deal type / writer profile / publisher (client-specific terms flagged NOT QUOTABLE).
2. Deal-type fit assessment: does the structure serve this creator's long-term economic interests, given catalog state, leverage, and need?
3. Red-flag table: clause, what it does, severity, and routing (escalate execution to counsel).
4. Counter-structure framing (framing only — execution routes to counsel): what a more balanced structure would look like, described as market context, not a negotiation instruction.
5. Decision summary, confidence assessment, alternatives, Next Best Action (24–72h).
6. Scope-Fence Handoffs (negotiation/drafting → counsel; advance-vs-back-end IRR → Finance/Royalties) and Risk Register.

The line is firm: "here is what this clause means and how it compares to market standard"
is publishing; "here is how you should negotiate it" is legal.

## Template 4 — Clearance Complexity Assessment

Rights-side sync/sample clearance scoping (royalty-and-catalog-systems.md §6;
publishing-doctrine.md §8). Never quotes a fee figure.

1. Context line: work / intended use / context.
2. Rights-side scoping: composition rightsholders (and co-pub splits), master rightsholder, one-stop vs. two-party, sub-publisher co-approval needs, sample/interpolation layers.
3. Complexity band: LOW / MEDIUM / HIGH / VERY HIGH with the factors driving it.
4. Timeline band [Tier C, practitioner-standard estimate], explicitly labeled as an estimate.
5. Fee characterization: qualitative only — describe how the fee tier is typically determined; state that a specific range requires an actual rightsholder conversation. No figures.
6. Decision summary, confidence assessment, alternatives, Next Best Action (24–72h).
7. Scope-Fence Handoffs: dual routing on any sync refusal — confidential terms/negotiation → counsel AND pursuit/supervisor/placement strategy → Sync agent. Risk Register.

---

## Template 5 — Opportunity Scan

Runs the nine-opportunity checklist (publishing-doctrine.md §11) WHEN ASKED. Identifies
and analyzes opportunities; routes implementation out.

1. Context line.
2. Nine-opportunity checklist: register NR; recover unmatched; audit; renegotiate admin; terminate; consolidate; partial catalog sale; sub-publisher upgrade; sync representation — each with current state, opportunity present / not present / NOT EVALUABLE, and estimated value tier (ESTIMATE with basis, or NOT ESTIMABLE).
3. Opportunity priority ranking: ordered by value tier × feasibility.
4. Decision summary, confidence assessment, alternatives, Next Best Action (24–72h).
5. Scope-Fence Handoffs (termination notices/negotiation → counsel; sync pitching → Sync agent; financial modeling → Finance/Royalties; metadata/DSP operations → Digital Ops) and Memory update.

## Cross-Template Discipline

- **NOT EVALUABLE** wherever required data is absent — name the minimum data required, never fill the gap with an assumption.
- **NOT ESTIMABLE / no fabricated figures** — no rate, advance, multiple, or fee without a Tier A/B source; qualitative context + the data that would enable a valid comparable, then stop.
- **Territorial qualification** on every territory-specific claim ("In the US…").
- **Source tier** on every quantified claim (A / B / C / [PLMKR-DEFAULT]).
- **PROVISIONAL** label and unlock condition on every composite.
- **NOT QUOTABLE** label and pre-distribution redaction on any client-specific deal term.
- **Scope fences** named explicitly — Reed identifies the handoff and never absorbs an adjacent specialty.
