# PLMKR Production Output Templates

Reusable output skeletons Beat fills in when producing each named output. PLMKR-owned original. Product-blind — no product names in any output. All unverified figures labeled **ESTIMATE / NOT QUOTABLE**; loudness/true-peak labeled **TARGET** until measured. Templates reference the Production Readiness Rubric; they do not restate full knowledge.

---

## Template 1 — Production Plan

**Production Plan — [Song Title] · [Artist] · v[X]** · Event target: [release date / submission] · Plan date: YYYY-MM-DD · Current phase: [pre-pro / tracking / mix / master / delivery]

- **Creative Direction Summary** — sonic target (genre · sub-genre · era · BPM range · dynamic character); Reference 1/2/3 (artist — title · specific element); instrumentation palette (named); brief status (Draft / Artist-approved in writing / Not yet created). *Every entry measurable or reference-anchored; genre-vibe shorthand is invalid without a named comparable.*
- **Production Team** — per role (producer, co-producer, mixer, mastering engineer, session players): name/status, ≥3 genre-matching credits, rate (ESTIMATE / NOT QUOTABLE until invoice-backed), deal form (points % + fee + advance; recoupment formula; WFH vs. royalty). *"TBD" is a gap; a deal without a stated ownership consequence is invalid output.*
- **Phase Schedule** — pre-pro / tracking / overdubs / mix / master / delivery; target start, hard deadline, key deliverable, owner, revision rounds built in, buffer. *Delivery buffer ≥2 weeks before release; a schedule without buffer assumes no revisions.*
- **Phase Budget** — budgeted (ESTIMATE) vs. invoice-backed actuals vs. variance + note; **contingency line mandatory (≥10% of subtotal)**; positive variance triggers a change-order entry.
- **Change-Order Log** — date, change, cost impact, schedule impact, quality tradeoff, approved by. *Every entry states all three of money/time/quality even when one is zero; a verbal change order is not one.*
- **Deliverables & Success Criteria** — stereo master (24-bit minimum; loudness/true-peak per current platform spec, labeled TARGET until measured — Dim-4 gate); stems (Dim-6 gate); instrumental; radio edit; ISRC; metadata; credits (artist-approved).
- **Decision Summary** (why / why now / supporting data / assumptions / what would change the answer) · **Confidence Assessment** (level, evidence basis, risks, confidence cap) · **Alternatives** (≥1 credible, or "none credible — [reason]") · **Next Best Action (24–72h)** (one concrete step).

## Template 2 — Production Readiness Scorecard

**Production Readiness Scorecard — [Song Title] · [Artist] · [Date]** · Rubric: Production Readiness Rubric · Scoring context: [pre-pro / tracking readiness / release-ready review]

- **Dimension Scores** — for each of the 8 dimensions: weight, score (1–10), evidence type (observed/told/inferred → MEASURED/SOURCED/JUDGED/AMBIGUOUS/ABSENT), confidence (high/medium/low), key assumption (changes score ≥1 if wrong). *Score each independently; told-absence of infrastructure is evidence (scored on anchor, not INFERRED); a missing track record on a first project is INFERRED at 3, never below.*
- **Hard Gate Flags** — Technical Quality Gate (Dim-4 = 1 → NOT DELIVERABLE) and Delivery & QC Gate (Dim-6 = 1 → RELEASE BLOCKED): CLEAR / TRIGGERED. *Gates are binary; do not use gate language for scores > 1.*
- **Composite** — weighted formula shown per dimension; append **PROVISIONAL** if any dimension is INFERRED (show its contribution separately).
- **Output Band** — GREEN 8.0–10.0 / YELLOW 6.0–7.9 / AMBER 4.0–5.9 / RED 1.0–3.9. *If either gate is TRIGGERED, state the gate verdict first — it overrides the band for the blocked verdict.*
- **Gap Summary & Remediation** — lowest-scoring dimensions: gap, remediation step, owner, target date.
- **Composite Confidence** (capped by the lowest-confidence critical dimension, named inline) · **Decision Summary** · **Alternatives** · **Next Best Action (24–72h)**.

## Template 3 — Budget & Schedule

**Budget & Schedule — [Song Title] · [Artist] · v[X]** · Scope: [single / EP / album — explicit] · Status: [Proposed / Approved / In-Progress / Final Reconciliation]

- **Scope Statement** — what is included and **explicitly excluded** (e.g., video, sync clearance, publishing registration, formats beyond stereo streaming master). *A budget without a scope statement cannot be assessed for overrun.*
- **Phase Budget Detail** — pre-pro / tracking (studio day rate, session players) / mix (fee + included revision rounds; additional rounds per round) / master / delivery / misc; subtotal; **contingency ≥10% (hard minimum)**; total. All figures ESTIMATE / NOT QUOTABLE until invoice-backed. Producer **advance** stated separately with recoupment mechanism.
- **Producer Deal Terms** — points (% of what base — state it), flat fee, advance + recoupment basis (royalties only / cross-collateralized), WFH vs. royalty per role, master ownership (label / artist / co-owned %), **co-write implication (Y/N — if Y, flag to Publishing, do not resolve here)**.
- **Budget Assumptions** — each assumption that, if wrong, changes the budget materially, labeled ESTIMATE / NOT QUOTABLE where unverified. **Change-Order Triggers** — pre-defined trigger list with the money/time/quality category named.
- **Schedule Summary** — phase start/end, key milestone, risk flag; release date ≥2 weeks after distributor delivery (flag if closer).
- **Decision Summary · Confidence Assessment · Alternatives · Next Best Action (24–72h).**

## Template 4 — Producer/Team Casting Brief

**Producer/Team Casting Brief — [Song Title] · [Artist] · v[X]** · Roles: [Producer / Mixer / Mastering Engineer / Session Players] · Context: [first-time / returning / one-off / full project]

- **Project Context** — genre/sub-genre (specific enough to filter credits by), sonic target summary, top-3 references (artist — title · defining element), artist career stage, release scope, approved budget **range** (ESTIMATE), timeline.
- **Per role — Match Criteria** — genre/sonic fit (≥3 credits for producer, ≥2 for mixer/master — artist/title/year, verified from release credits), career-stage alignment, rate range (ESTIMATE), deal structure preference (state ownership consequence), collaboration history, availability (confirm in writing).
- **Casting Red Flags** — rate >30% over budget with no resolution path; credits not matching the genre/sonic tier; prior unresolved disputes; ownership terms misaligned with project structure; no documented credit at the sonic tier with a measurable outcome.
- **Preferred Candidate(s)** + **Recommendation** with the specific match rationale (genre credit alignment, rate fit, stage fit, deal-structure fit). If no candidate meets criteria, say so and name the gap.
- **Deal Structure Summary** — per role: preferred structure, ownership consequence, recoupment consequence; **session players WFH confirmed in writing** before tracking; **co-write implication flagged and referred to Publishing** if production triggers a composition split.
- **Decision Summary · Confidence Assessment · Alternatives · Next Best Action (24–72h).**

> A Casting Brief with signed deal memos is the observed evidence for a Dim-2 score above 7 and a Dim-8 score above 5; entries that are reconciled but not yet signed score Dim-8 at most 5.

## Template 5 — Delivery & QC Checklist

**Delivery & QC Checklist — [Song Title] · [Artist] · [Date]** · Release target: [streaming / download / physical — no platform names] · Deadline: YYYY-MM-DD · Checked by: [role]

*Operational completeness gate — carries Confidence Assessment (a release-ready verdict is a claim) but not Alternatives / Next Best Action.*

- **Gate Status** — Technical Quality Gate (Dim-4: loudness + format + stem spec) and Delivery & QC Gate (Dim-6: full package present and verified): CLEAR / TRIGGERED / PENDING. *A TRIGGERED gate blocks the verdict regardless of every other item.*
- **A. Master File** — format (24-bit minimum; sample rate per spec); integrated loudness (TARGET until measured); true-peak (≤ ceiling, TARGET until measured); dynamic range; no clipping/artifacts; file integrity; mastering sign-off documented.
- **B. Stems Package** — required? (state basis); list matches spec; same format/sample rate as master; naming matches spec exactly; stems sum to master; all present.
- **C. Instrumental** / **D. Radio Edit** — required? (state basis); lead vocal fully removed / edit points clean; format and loudness match master; labeled correctly.
- **E. Metadata** — title, primary artist, featuring, **ISRC (embedded in the file)**, UPC/EAN, release date, genre/sub-genre/mood, songwriter/composer (full legal names), publisher, producer/mixer/master credits, **P-line (master owner) and C-line (composition owner) — distinct fields**, explicit flag, language.
- **F. Credits** — written credits sheet; **artist-approved in writing**; session-player WFH confirmed; credits match metadata exactly; **uncleared samples flagged to legal — hard block on distribution**.
- **G. Distributor QC** — submitted (date); QC pass/fail/pending; flags raised and resolved; delivery confirmed received.
- **Overall Package Status** — READY (all checked, both gates CLEAR, distributor QC clean) / PENDING (list outstanding items) / BLOCKED (a gate TRIGGERED, uncleared samples, or QC failed). **Confidence Assessment** with cap named.
