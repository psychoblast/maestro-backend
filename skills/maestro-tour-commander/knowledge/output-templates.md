# PLMKR Tour & Live — Output Templates

Four structured output templates for the Tour & Live function. Every template
enforces the same discipline:

- **Mandatory core (every output):** Work Product · Decision Summary (why / why
  now / what data / what assumptions / what would change the answer) · Confidence
  Assessment with the weakest-link cap named.
- **Decision-type fields (all four templates):** Alternatives · Next Best Action (24–72h).
- **Conditional modules (attach only when relevant):** Scores · Risk Register.

Rules carried into every template:
- **NOT EVALUABLE:** when required data is absent, state NOT EVALUABLE, name the
  data required, and stop. No figures or recommendations without basis.
- **NOT FABRICABLE:** no guarantee benchmark, production-cost estimate, or
  specific touring figure without a Tier A/B source. Describe the qualitative
  factors and name the documented data that would enable a valid comparable —
  stop before the number.
- **Probability:** qualitative confidence bands (HIGH / MEDIUM / LOW with
  reasons); no numeric probability percentages pre-unlock.
- **Scope fence:** name the handoff; never absorb the adjacent specialty. Every
  template includes scope-fence handoffs.

## Template 1 — Tour Routing Assessment

**When to use:** evaluating a proposed or draft routing — anchor selection,
market sequence, dead-leg analysis, routing efficiency, break-even alignment.
Use the rubric for Dimensions 1 (Routing Logic) and 2 (Financial Model Integrity).

```
# TOUR ROUTING ASSESSMENT — [Artist / Tour Name]
Date: [YYYY-MM-DD]
Evaluation context: [pre-announcement review / post-commitment / ongoing monitoring]

## WORK PRODUCT
### Routing Summary
- Total dates: [count] · Market-tier distribution: [breakdown]
- Proposed sequence: [list in order]
- Anchor dates confirmed: [yes/partial/no — list anchors + status]
- Dead legs identified: [City A → City B, distance, no show]

### Anchor-First Assessment
[Were anchors identified and confirmed before secondary dates? Evidence + verdict.]

### Dead Leg Analysis
[Per dead leg: travel cost vs. fill-market opportunity. Flag negative net contribution.]
[NOT EVALUABLE if transportation cost data is absent — name the data required.]

### Routing Efficiency Ratio
- Show days ÷ total tour days = [ratio] — [≥0.65 efficient · 0.55–0.65 adequate · <0.55 flag]

### Market Sequence Alignment
[Does the sequence align with streaming / live-draw data by market?
If absent: NOT EVALUABLE for sequence assessment.]

### Break-Even Alignment Check
[Does a break-even model exist? If NOT: HG-1 FAIL — flag immediately.
If yes: does the guarantee total meet the break-even floor?]

## RUBRIC SCORES
- Dimension 1 — Routing Logic: [Grade] — [MEASURED/SOURCED/JUDGED evidence]
- Dimension 2 — Financial Model Integrity: [Grade] — [evidence]
- HG-1 (Break-Even Before Commitment): [PASS/FAIL/NOT EVALUABLE]
- HG-2 (Anchor Before Fill): [PASS/FAIL/NOT EVALUABLE]
- PROVISIONAL COMPOSITE (D1+D2): [numeric] — unlock: ≥30 outcome-checked evaluations

## DECISION SUMMARY
Recommendation: [PROCEED / PROCEED WITH MODIFICATIONS / HOLD / REJECT]
1. Why?  2. Why now?  3. What data supports this?  4. What assumptions?  5. What would change my answer?

## CONFIDENCE ASSESSMENT
Level: [HIGH/MEDIUM/LOW] · Reasons · Risks · Weakest-link cap: [named]

## RISK REGISTER
| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|

## ALTERNATIVES
[At least one credible alternative routing, or "none credible — [reason]".]

## NEXT BEST ACTION (24–72h)
[Single concrete step.]

## SCOPE-FENCE HANDOFFS
| Topic | Routes to |
|-------|-----------|
| Legal review of routing deal memos | LEX-CIPHER + counsel |
| Detailed financial modeling beyond break-even structure | the finance function |
| Marketing campaign for tour announcement | the marketing function |
```

## Template 2 — Offer Evaluation Report

**When to use:** evaluating a specific live offer — guarantee, structure,
backend, production provisions, marketing commit, radius/exclusivity. Requires
the current deal memo / offer letter; if absent: NOT EVALUABLE for red-flag
clause assessment.

```
# OFFER EVALUATION REPORT — [Artist / Market / Venue]
Date: [YYYY-MM-DD]
Offer source: [promoter / venue]

## WORK PRODUCT
### Offer Summary
- Market · Venue (tier) · Capacity · Offered guarantee · Deal structure
- Split point (if any) · Backend % · Promoter expense deductions
- Marketing commitment [or "not specified — flag"] · Production provisions
- Exclusivity / radius clause · Offer expiry

### Deal Structure Assessment
[Flat / guarantee+backend / split-point / four-wall / co-headline — which is
appropriate for this artist at this venue tier, and why?]

### Split-Point Reachability
[At average price × expected attendance, is the split point reached?
≥85% capacity required → flag backend LOW-PROBABILITY.
If capacity/price data absent: NOT EVALUABLE.]

### Break-Even Check
[Does the guarantee cover this show's allocated break-even cost?
If no tour break-even model: HG-1 flag — NOT EVALUABLE for an acceptance call.]

### Marketing Commitment
[Specified? If not: RED FLAG — require specification before signing.]

### Red-Flag Clause Table
| Clause | Present? | Risk | Recommended Action |
|--------|----------|------|--------------------|
| Uncapped production deductions | | | [negotiate cap / route to LEX-CIPHER] |
| Marketing deductions without receipts | | | [require receipts] |
| Post-settlement clawback | | | [refuse / route to LEX-CIPHER] |
| Comp allotment undefined | | | [define in deal memo] |
| Settlement timing undefined | | | [specify in deal memo] |
| Exclusivity / radius wider than expected | | | [evaluate routing impact] |

## DECISION SUMMARY
Recommendation: [ACCEPT / ACCEPT WITH CONDITIONS / COUNTER-PROPOSE / DECLINE]
1. Why?  2. Why now?  3. Data?  4. Assumptions?  5. What would change my answer?

## CONFIDENCE ASSESSMENT
Level · Reasons · Risks · Weakest-link cap

## RISK REGISTER
| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|

## ALTERNATIVES
[Alternative deal structure, e.g. a higher flat guarantee with no backend.]

## NEXT BEST ACTION (24–72h)

## SCOPE-FENCE HANDOFFS
| Topic | Routes to |
|-------|-----------|
| Red-flag clause legal review / negotiation | LEX-CIPHER + counsel |
| Financial P&L modeling for this show | the finance function |

```
# TOUR P&L PRE-MODEL — [Tour / Artist]
Date: [YYYY-MM-DD]
Model status: PRE-COMMITMENT (models proposed routing; not final guarantees)

## WORK PRODUCT

|--------|--------------|--------|-------|

| Backend receipts | $0 | | only model with documented basis |
| VIP / meet-and-greet | | | based on [prior data / NOT EVALUABLE] |
| Merch (net of hall fees) | | | [Tier C; verify hall fees per venue] |
[Cite Tier A/B/C per estimate; flag NOT EVALUABLE where absent.]

### Expense Projection
| Category | Amount | Basis |
|----------|--------|-------|
| Agent commission (10–15%, Tier C) | | |
| Management commission (15–20%, Tier C) | | |

| Accommodation · Crew wages · TM fee | | |
| Per diems ($50–$100/day, Tier C) · Insurance | | |

### Break-Even Analysis

Festival: [name + date] · Slot offered: [stage, day, set time, length]

## WORK PRODUCT
### Festival Profile
- Tier · Audience-demographic alignment [HIGH/MEDIUM/LOW + rationale]
- Booking lead time vs. campaign timing · Prior relationship

### Slot Assessment
- Stage / day / time · Career-stage alignment · Adjacent acts (discovery effect)

### Release Campaign Alignment
[Active campaign timed to this festival? Assess amplification or standalone value.]

- Territory (radius / named markets) · Time window (days before/after)
- Conflicts with confirmed or planned shows

  [NOT EVALUABLE if no routing plan exists for comparison]

## DECISION SUMMARY

|------|-----------|--------|-----------|

## ALTERNATIVES
